# Daily Ledger — دفتر روزانه تسک‌ها

سایت مدیریت تسک‌های روزانه پروژه‌های مختلف (توییت، دیسکورد، و هرچی تعریف کنی)، با پنل مدیریتی، چک‌لیست روزانه، آمار، دو زبانه (فارسی/انگلیسی)، دارک/لایت، و ورود با گوگل.

## استک

Next.js 14 (App Router) · Supabase (Auth + Postgres) · Tailwind · next-intl · Recharts

---

## اگه قبلاً schema.sql رو اجرا کرده بودی

فقط کافیه `supabase/migration_002.sql` رو هم توی SQL Editor اجرا کنی — ستون‌ها و جدول‌های جدید (یوزرنیم، دسته‌بندی تسک، ایموجی، آرشیو، ایردراپ‌ها) رو بدون پاک کردن دیتای قبلی اضافه می‌کنه.

## ورود با ایمیل/رمز

توی Supabase چون قرار شد نیازی به تایید ایمیل نباشه، برو به:
```
Authentication → Sign In / Providers → Email
```
و گزینه‌ی **Confirm email** رو خاموش کن. با این کار signUp بلافاصله بدون نیاز به کلیک روی لینک ایمیل، کاربر رو وارد می‌کنه.

## مرحله ۱ — دیتابیس روی Supabase

1. وارد پروژه‌ای که قبلاً توی [supabase.com](https://supabase.com) ساختی بشو.
2. برو به **SQL Editor** و کل محتوای فایل `supabase/schema.sql` رو اجرا کن. این کار جدول‌ها، Row Level Security (که دیتای هر کاربر رو فقط برای خودش قابل دیدن می‌کنه)، و trigger ساخت خودکار پروفایل رو می‌سازه.
3. از **Project Settings → API** دو مقدار رو بردار: `Project URL` و `anon public key`. این‌ها رو بعداً لازم داری.

## مرحله ۲ — فعال‌سازی ورود با گوگل

1. توی Supabase برو به **Authentication → Providers → Google** و فعالش کن.
2. `Client ID` و `Client Secret` که از Google Cloud Console گرفتی رو همون‌جا وارد کن.
3. توی Google Cloud Console (همون OAuth Consent Screen / Credential که ساختی)، زیر **Authorized redirect URIs** این آدرس رو اضافه کن (از خود صفحه Google provider توی Supabase کپی می‌شه، یه چیزی شبیه این):
   ```
   https://YOUR_PROJECT.supabase.co/auth/v1/callback
   ```
4. بعد از دیپلوی روی Vercel، دامنه نهایی سایتت (مثلاً `https://your-app.vercel.app`) رو هم به **Authentication → URL Configuration → Redirect URLs** توی Supabase اضافه کن.

## مرحله ۳ — اجرای محلی (اختیاری، برای تست قبل از دیپلوی)

```bash
npm install
cp .env.example .env.local
# مقدار دو متغیر رو توی .env.local با مقادیر خودت پر کن
npm run dev
```

سایت روی `http://localhost:3000` بالا میاد.

## مرحله ۴ — پوش کردن روی گیت‌هاب

```bash
git init
git add .
git commit -m "Initial commit"
gh repo create daily-ledger --private --source=. --push
# یا دستی: یه ریپو بساز و origin رو ست کن و push بزن
```

## مرحله ۵ — دیپلوی روی Vercel

1. با اکانت گیت‌هابت وارد [vercel.com](https://vercel.com) شو و **New Project** بزن.
2. ریپوی `daily-ledger` رو انتخاب کن.
3. زیر **Environment Variables** این دو تا رو اضافه کن (همونایی که از Supabase برداشتی):
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. Deploy بزن. بعد از اولین دیپلوی، دامنه نهایی رو بردار و طبق مرحله ۲ (بند ۴) به Supabase اضافه کن، بعد یه بار دیگه از Vercel **Redeploy** بزن.

## ساختار پروژه

```
app/[locale]/(app)/       صفحات پشت لاگین: داشبورد، مدیریت، آمار، تنظیمات
app/[locale]/login/       صفحه ورود
app/auth/callback/        بازگشت از OAuth گوگل
components/                کامپوننت‌های UI
lib/recurrence.ts          موتور تشخیص «امروز چه تسک‌هایی باید انجام بشه»
lib/stats.ts                محاسبه انجام‌شده/میس‌شده و streak
lib/dates.ts                فرمت تاریخ شمسی/میلادی و تایم‌زون
supabase/schema.sql          کل اسکیمای دیتابیس
messages/fa.json, en.json    متن‌های دو زبانه
```

## نکات مهم درباره منطق برنامه

- **«میس‌شده» چطور محاسبه می‌شه؟** هیچ Cron Job ای لازم نیست. هر تسکی که تاریخ سررسیدش گذشته و لاگ `done` نداره، همون لحظه که صفحه آمار یا داشبورد رو باز می‌کنی به‌صورت محاسبه‌ای (نه ذخیره‌شده) «میس‌شده» حساب می‌شه. یعنی داده همیشه با تایم‌زون خودت به‌روزه.
- **Streak (روزهای پیاپی)** یعنی چند روز پشت‌سرهم همه‌ی تسک‌های اون روز رو کامل انجام دادی؛ روزهایی که هیچ تسکی سررسید نداشته streak رو نمی‌شکنن.
- هر کاربر با گوگل که وارد بشه، به‌خاطر Row Level Security فقط دیتای خودش رو می‌بینه و می‌سازه — نیازی به هیچ تنظیم اضافه نیست.

## چی جدید توی این آپدیت اضافه شد

- ورود/ثبت‌نام با ایمیل و رمز عبور، در کنار گوگل
- انتخاب یوزرنیم (موقع ثبت‌نام با ایمیل، یا اولین ورود برای کاربرهای گوگل — یه فرم کوچیک قبل از ورود به سایت نشون داده می‌شه)
- مسکات دخترک انیمیشنی اورجینال (طراحی خودمون، نه کاراکتر کپی‌رایتی) که با اسم کاربر خوش‌آمد می‌گه — هم توی داشبورد، هم صفحه ورود
- بنر بالا حالا بین تاریخ/ساعت و نقل‌قول‌های انگیزشی (همیشه هم انگلیسی هم فارسی با هم) می‌چرخه؛ توی حالت فارسی هر دو تاریخ شمسی و میلادی نشون داده می‌شه
- ساعت مینیمال بزرگ‌تر شده و قابل درگ‌کردن به هر جای صفحه‌ست (موقعیتش توی مرورگر ذخیره می‌شه)
- چک‌لیست روزانه: هر تسک با ایموجی و دسته خودش، بعد از تیک زدن با انیمیشن محو می‌شه و میره به بخش «امروز انجام شد»، شمارنده کارهای باقی‌مونده بالای صفحه
- پنل مدیریت: دسته‌بندی آماده برای کارهای کریپتویی (تراکنش، لیکوییدیتی، دیلی چک، دیسکورد، توییت، محتوا) + دسته سفارشی، انتخاب ایموجی، آرشیو/بازگردانی، کپی تسک، جستجو و فیلتر
- بخش «ایردراپ‌ها» برای ثبت دراپ‌های دریافتی
- تنظیمات: یوزرنیم، روز شروع هفته، خاموش/روشن کردن نقل‌قول‌ها، خروجی CSV، حذف کامل داده‌ها

## قابلیت‌هایی که بعداً می‌شه اضافه کرد

- اعلان مرورگر (Web Push) برای یادآوری تسک‌های انجام‌نشده نزدیک پایان روز
- نصب به‌عنوان PWA روی موبایل
- ایمیل خلاصه هفتگی (با Resend)
- حذف کامل حساب کاربری (الان فقط داده‌ها پاک می‌شن؛ حذف خود اکانت auth نیاز به service role key داره که به‌خاطر امنیت سمت کلاینت نذاشتیم)
