# راه‌اندازی بات تلگرام

این یه راهنمای فنیه، جدا از README (که فقط معرفی سایته). فقط یه‌بار لازمه انجامش بدی.

## این بخش دقیقاً چیکار می‌کنه

هیچ پیامی خودکار و بدون خواست کاربر فرستاده نمی‌شه. هر کاربر خودش، روی هر تسکی که بخواد، می‌تونه بگه «۵ ساعت دیگه»، «هر روز ساعت ۲۱:۰۰»، یا «هر ۴ ساعت» یادآوریش کنه — یا هیچی نگه (پیش‌فرض خاموشه). اگه تسک قبل از موعد یادآوری تیک خورده باشه، پیامی هم فرستاده نمی‌شه.

## ⚠️ اگه دیپلوی ورسل ارور می‌داد

دلیلش این بود: تو `vercel.json` یه زمان‌بندی هر ۱۵ دقیقه گذاشته بودم، ولی پلن Hobby ورسل فقط زمان‌بندی **روزی یک‌بار** رو قبول می‌کنه و هر چیز پرتکرارتر رو همون موقع دیپلوی رد می‌کنه. این مشکل رفع شد — `vercel.json` الان روزی یه‌بار (به‌عنوان یه پشتیبان کلی) اجرا می‌شه و دیگه دیپلوی رو خراب نمی‌کنه. برای یادآوری‌های دقیق‌تر، مرحله‌ی ۶ رو (پایین) دنبال کن.

## مرحله ۱ — اجرای migration دیتابیس

توی Supabase → SQL Editor، این فایل‌ها رو (هرکدوم که قبلاً نزدی) اجرا کن:

1. `supabase/migration_008_telegram_announcement_settings.sql`
2. `supabase/migration_009_task_reminders.sql`
3. `supabase/migration_011_task_inbox.sql` (اگه امکان کابینت رو هم فعال کردی)
4. `supabase/migration_012_telegram_bot.sql` ← **این یکی لازمه** برای منوی جدید بات (ساخت پروژه/تسک از تلگرام) و بخش انتقادات و پیشنهادات
5. `supabase/migration_013_telegram_bot_v2.sql` ← **این هم لازمه** برای ارسال همگانی/ریست/ویرایش متن منو از داخل تلگرام

## مرحله ۲ — ساخت بات تو تلگرام

1. تو تلگرام برو سراغ [@BotFather](https://t.me/BotFather).
2. بفرست: `/newbot`
3. یه اسم و یه یوزرنیم (باید به `bot` ختم بشه، مثلاً `daily_ledger_bot`) بهش بده.
4. یه **توکن** بهت می‌ده — نگه دار.

## مرحله ۳ — گرفتن Service Role Key از Supabase

توی Supabase برو **Project Settings → API** و مقدار **service_role** رو بردار (نه anon key — این یکی حساسه، فقط تو Environment Variables سرور بذارش).

## مرحله ۴ — متغیرهای محیطی روی Vercel

توی **Vercel → Project Settings → Environment Variables**:

| متغیر | مقدار |
|---|---|
| `TELEGRAM_BOT_TOKEN` | توکنی که از BotFather گرفتی |
| `NEXT_PUBLIC_TELEGRAM_BOT_USERNAME` | یوزرنیم بات بدون `@` |
| `TELEGRAM_WEBHOOK_SECRET` | یه رشته‌ی تصادفی (مثلاً `openssl rand -hex 20`) |
| `CRON_SECRET` | یه رشته‌ی تصادفی دیگه |
| `SUPABASE_SERVICE_ROLE_KEY` | همون service_role که از Supabase برداشتی |

بعدش یه بار **Redeploy** بزن.

## مرحله ۵ — وصل کردن وبهوک بات به سایت

این آدرس رو (با مقادیر خودت) توی مرورگر باز کن:

```
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/setWebhook?url=https://<YOUR_DOMAIN>/api/telegram/webhook&secret_token=<TELEGRAM_WEBHOOK_SECRET>
```

جواب `{"ok":true,...}` یعنی موفق بود. از این مرحله به بعد، دکمه‌ی «اتصال به تلگرام» تو تنظیمات سایت کار می‌کنه.

## مرحله ۶ — یادآوری‌های دقیق (پیشنهاد: GitHub Actions، رایگان و بدون محدودیت روز)

پلن Hobby ورسل فقط اجازه‌ی چک کردن روزی یک‌بار رو می‌ده که برای «۵ ساعت دیگه یادآوری کن» دقیق نیست. به‌جای یه سرویس بیرونی که ممکنه بعد از مدتی غیرفعال بشه (بعضی از این سرویس‌ها بعد از چند بار خطای پشت‌سرهم خودشون Cron رو خاموش می‌کنن)، از **GitHub Actions** استفاده می‌کنیم — کاملاً رایگان، همون گیت‌هاب خودت، بدون هیچ محدودیت روز یا تعداد اجرا.

یه فایل آماده تو پروژه هست: `.github/workflows/telegram-reminders.yml` — هر ۱۵ دقیقه خودکار endpoint یادآوری‌ها رو صدا می‌زنه. فقط باید دو تا Secret به ریپوی گیت‌هابت اضافه کنی:

1. برو به ریپوت تو گیت‌هاب → **Settings → Secrets and variables → Actions → New repository secret**
2. دوتا secret بساز:
   - `APP_URL` → آدرس کامل سایتت، مثلاً `https://your-app.vercel.app` (بدون `/` آخر)
   - `CRON_SECRET` → همون مقداری که رو Vercel ست کردی
3. همین! از دفعه‌ی بعدی که پوش می‌کنی (یا از تب **Actions** تو گیت‌هاب دستی روشنش کن) شروع به کار می‌کنه.

**یه نکته:** گیت‌هاب اگه یه ریپو ۶۰ روز هیچ کامیتی نگیره، Workflow‌های زمان‌بندی‌شده‌ش رو خودکار متوقف می‌کنه. اگه یه مدت طولانی سراغ پروژه نری و بعد دیدی یادآوری‌ها نمیان، کافیه بری تب **Actions** تو گیت‌هاب و دوباره روشنش کنی (یا فقط یه کامیت کوچیک بزن).

## منوی بات — ساخت پروژه، ثبت تسک، کابینت

بعد از وصل شدن (همون مرحله‌ی /start همراه با کد)، بات یه منوی کامل با دکمه نشون می‌ده و یه لیست دستور اسلش هم کنار جعبه‌ی پیام داره (همون سه‌خط معروف بات‌ها):

- **📁 پروژه‌های من** — پروژه‌هات رو می‌بینی، یا یکی جدید می‌سازی. روی هر پروژه که بزنی، دکمه‌ی «افزودن تسک» میاد.
- افزودن تسک مرحله‌به‌مرحله می‌پرسه: اسم تسک → توضیحات (یا رد کن) → لینک (یا رد کن) → یه خلاصه نشون می‌ده و تأیید نهایی می‌گیره.
- هر تسکی که از تلگرام ساخته بشه، خودکار با دسته‌بندی «ارسالی از تلگرام» (`telegram_push`) و تکرار روزانه ثبت می‌شه — همون روز توی «امروز»ِ سایت ظاهر می‌شه.
- **📥 افزودن از لینک اشتراکی** — وقتی یکی از دوستات از سایت براش تسک فرستادی و لینک `/shared/...` رو بهش دادی، کافیه همون لینک رو تو بات پیست کنه — خودکار همه‌ی تسک‌های اون بسته میره توی **کابینتش** (همونی که قبلاً امانتی‌ها صداش می‌کردیم)، بدون اینکه لازم باشه لینک رو تو مرورگر باز کنه.
- **مهم:** انجام دادن (تیک زدن) تسک‌های روزانه هنوزم فقط از توی خود سایت انجام می‌شه؛ بات فقط یه راه اضافه برای **ثبت** تسکه.
- **📮 انتقاد و پیشنهاد** — هر پیامی که کاربر اینجا بفرسته، با یوزرنیمش مستقیم برای همه‌ی مدیرهایی که تلگرامشون وصله فرستاده می‌شه. مدیر کافیه روی همون پیام توی تلگرام **Reply** بزنه — جوابش خودکار برای همون کاربر می‌ره، و بعدش هم پیام اصلی هم ریپلای مدیر خودکار از چت مدیر پاک می‌شه که شلوغ نمونه.
- دکمه‌ی زبان هم هست؛ بات هم فارسی جواب می‌ده هم انگلیسی (بسته به انتخاب خودت).

نکته: برای اینکه مدیر بتونه جواب انتقادات رو بگیره، خودش هم باید مثل بقیه از تنظیمات سایت یه کد بگیره و حساب تلگرامش رو وصل کنه. مدیر بودن یعنی همون ایمیلی که مالک سایته، یا هر ایمیلی که از تب Admins اضافه شده.

## دستورات اسلش (/) — هم برای کاربر عادی، هم مدیر

این‌ها هم از منوی دکمه‌ای در دسترسن، هم می‌شه مستقیم تایپ‌شون کرد:

| دستور | برای کی | کارش چیه |
|---|---|---|
| `/start` | همه | شروع / اتصال حساب با کد |
| `/menu` | همه | نمایش دوباره‌ی منوی اصلی |
| `/feedback` | همه | میان‌بر مستقیم برای فرستادن انتقاد و پیشنهاد |
| `/cancel` | همه | لغو هر مرحله‌ای که وسطشی |
| `/stop` | همه | قطع اتصال حساب از بات |
| `/broadcast [متن]` | فقط مدیر | ارسال پیام به همه‌ی کاربرهایی که تلگرامشون وصله (با تأیید نهایی) |
| `/reset_me` | فقط مدیر | ریست کردن بات فقط برای خود مدیر (اگه گیر کرد) |
| `/reset_user یوزرنیم` | فقط مدیر | ریست کردن بات برای یه کاربر خاص با یوزرنیم تلگرامش |
| `/reset_all` | فقط مدیر | ریست کردن بات برای **همه‌ی** کاربرها یکجا (با تأیید نهایی) |

«ریست» یعنی مکالمه‌ی نیمه‌کاره‌ی اون چت (مثلاً وسط ثبت یه تسک گیر کرده بود) پاک می‌شه و برمی‌گرده به حالت اول — دیگه لازم نیست کاربر خودش `/stop` و دوباره `/start` بزنه.

از منوی اصلی، مدیر همچنین دو تا دکمه‌ی اضافه می‌بینه: **📢 ارسال پیام به همه** (همون کار `/broadcast`) و **✏️ ویرایش متن منو** — که می‌تونه بدون نیاز به دیپلوی جدید، متن خوش‌آمدگویی منوی اصلی بات رو (جدا برای فارسی و انگلیسی) از همون تلگرام عوض کنه. با دکمه‌ی «برگردوندن به پیش‌فرض» هم می‌شه هر وقت خواست برگردوندش به حالت اول.
