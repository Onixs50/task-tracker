# راه‌اندازی بات تلگرام

این یه راهنمای فنیه، جدا از README (که فقط معرفی سایته). فقط یه‌بار لازمه انجامش بدی.

## این بخش دقیقاً چیکار می‌کنه

هیچ پیامی خودکار و بدون خواست کاربر فرستاده نمی‌شه. هر کاربر خودش، روی هر تسکی که بخواد، می‌تونه بگه:

- «۵ ساعت دیگه یادآوریم کن»
- «هر روز ساعت ۲۱:۰۰ یادآوریم کن»
- «هر ۴ ساعت یادآوریم کن»

یا اصلاً هیچی نگه (پیش‌فرض خاموشه). اگه اون تسک همون روز قبل از موعد یادآوری تیک خورده باشه، پیامی هم فرستاده نمی‌شه. تو به‌عنوان مدیر سایت هم می‌تونی جدا یه پیام دستی برای همه‌ی وصل‌شده‌ها بفرستی (از صفحه‌ی «مدیریت سایت»).

## مرحله ۱ — اجرای migration دیتابیس

توی Supabase → SQL Editor، به ترتیب این دو فایل رو اجرا کن (اگه قبلاً migration_008 رو زدی، فقط همین یکی کافیه):

1. `supabase/migration_008_telegram_announcement_settings.sql`
2. `supabase/migration_009_task_reminders.sql`

## مرحله ۲ — ساخت بات تو تلگرام

1. تو تلگرام برو سراغ [@BotFather](https://t.me/BotFather).
2. بفرست: `/newbot`
3. یه اسم و یه یوزرنیم (باید به `bot` ختم بشه، مثلاً `daily_ledger_bot`) بهش بده.
4. یه **توکن** بهت می‌ده، یه چیزی شبیه `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` — اینو نگه دار.

## مرحله ۳ — گرفتن Service Role Key از Supabase

توی Supabase برو **Project Settings → API** و مقدار **service_role** رو بردار (نه anon key — این یکی خیلی حساسه، هیچ‌جا جز Environment Variables سرور نذارش).

## مرحله ۴ — متغیرهای محیطی روی Vercel

توی **Vercel → Project Settings → Environment Variables** این‌ها رو اضافه کن:

| متغیر | مقدار |
|---|---|
| `TELEGRAM_BOT_TOKEN` | همون توکنی که از BotFather گرفتی |
| `NEXT_PUBLIC_TELEGRAM_BOT_USERNAME` | یوزرنیم بات بدون `@` (مثلاً `daily_ledger_bot`) |
| `TELEGRAM_WEBHOOK_SECRET` | یه رشته‌ی تصادفی دلخواه بساز (مثلاً با `openssl rand -hex 20`) |
| `CRON_SECRET` | یه رشته‌ی تصادفی دیگه بساز |
| `SUPABASE_SERVICE_ROLE_KEY` | همون service_role که از Supabase برداشتی |

بعد از اضافه کردن، یه بار **Redeploy** بزن.

## مرحله ۵ — وصل کردن وبهوک بات به سایت

بعد از دیپلوی، این آدرس رو (با مقادیر خودت جایگزین‌شده) توی مرورگر باز کن یا با curl بزن:

```
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/setWebhook?url=https://<YOUR_DOMAIN>/api/telegram/webhook&secret_token=<TELEGRAM_WEBHOOK_SECRET>
```

اگه جواب `{"ok":true,"result":true,...}` گرفتی یعنی وصل شد. از این مرحله به بعد، اتصال حساب کاربرها (دکمه‌ی «اتصال به تلگرام» تو تنظیمات) کار می‌کنه.

## مرحله ۶ — راه‌اندازی زمان‌سنج ارسال یادآوری‌ها

چون هر یادآوری یه زمان دقیق داره («۵ ساعت دیگه»، «ساعت ۲۱:۰۰»، …)، یه چیزی لازمه هر چند دقیقه یه‌بار چک کنه ببینه کدوم یادآوری الان سررسیدشه و بفرستدش — این فقط یه مکانیزم تحویله، نه چیزی که خودش تصمیم بگیره به کسی پیام بده.

**اگه پلن Vercel‌ت Pro باشه:** فایل `vercel.json` از قبل تنظیم شده که هر ۱۵ دقیقه این کار رو انجام بده — کار دیگه‌ای لازم نیست.

**اگه پلن Vercel‌ت Hobby (رایگان) باشه:** Vercel روی این پلن Cron Job رو فقط یه‌بار در روز اجرا می‌کنه، که برای یادآوری‌های ساعتی/چند-ساعته دقیق نیست. راه‌حل رایگان و ساده: از یه سرویس بیرونی مثل cron-job.org (رایگان) استفاده کن:

1. یه حساب بساز.
2. یه Cron Job جدید بساز که هر ۱۵ دقیقه این آدرس رو با متد GET صدا بزنه:
   ```
   https://<YOUR_DOMAIN>/api/cron/telegram-reminders
   ```
3. تو تنظیمات همون Cron Job، یه HTTP Header اضافه کن:
   ```
   Authorization: Bearer <CRON_SECRET>
   ```
   (همون مقداری که برای CRON_SECRET تو Vercel ست کردی)

همین. از این به بعد یادآوری‌ها با تاخیر حداکثر ۱۵ دقیقه‌ای سر وقت می‌رسن.

## تست کردن

بدون این تنظیمات هم کل بقیه‌ی سایت عادی کار می‌کنه — این بخش کاملاً اختیاریه و فقط وقتی متغیرهای بالا ست بشن فعال می‌شه.
